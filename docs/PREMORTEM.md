## 💀 시나리오 1: 금융 데이터 연동 실패

**가상의 실패 원인**  
카드사·보험사·통신사 API 연동이 예상보다 훨씬 복잡했다.  
오픈뱅킹으로 일부 데이터는 가져올 수 있었지만,  
보험료·학원비·통신비의 실시간 연동은 불가했고,  
사용자가 수동 입력을 거부하면서 "자동화"의 핵심 가치가 무너졌다.

**리스크 수준:** 🔴 High  
**대응책:**
- MVP 단계에서 수동 입력 + 영수증 OCR로 우선 시작
- 카드사 오픈뱅킹(마이데이터) 연동을 1순위 기술 과제로 설정
- 파트너십 우선 대상: 하나·신한 마이데이터 API

---

## 💀 시나리오 2: "굳이 왜 이 앱을" — 낮은 전환율

**가상의 실패 원인**  
"고정비 줄이기"에 공감하지만 앱을 설치하고 계정을 만들 만큼  
불편함을 느끼지 못하는 사용자가 많았다.  
다운로드 후 첫 주 내 이탈률 72%. 유지 사용자 거의 없음.

**리스크 수준:** 🔴 High  
**대응책:**
- 온보딩 3분 내에 "지금 당장 절감 가능한 금액"을 보여주는 훅 설계
- 첫 화면에서 평균 절감 금액 (예: "사용자 평균 월 127,000원 절감") 노출
- 카카오 공유 기능: "우리 가족 고정비 진단 결과 공유하기"

---

## 💀 시나리오 3: 투자 기능의 법적 리스크

**가상의 실패 원인**  
Layer 4의 AI 포트폴리오 추천이 금융투자업 무허가 영업으로 금감원 제재.  
"ETF 추천"이 투자 자문으로 해석되어 서비스 중단 처분.

**리스크 수준:** 🔴 High  
**대응책:**
- 법적 검토를 MVP 개발 전에 완료 (금융투자업 vs. 투자 정보 제공 경계 확인)
- 초기에는 "추천" 대신 "정보 제공" + "시뮬레이션" 형태로 설계
- 실제 투자 실행은 기존 증권사 앱으로 딥링크 연결 (직접 매매 기능 없음)
- 투자 성향 진단 결과에 "이 정보는 투자 권유가 아닙니다" 고지 명시

---

## 💀 시나리오 4: 개인정보 보안 사고

**가상의 실패 원인**  
가족 전체의 금융 정보를 집약한 탓에 데이터 유출 시 피해 규모가 컸다.  
해킹 사고 한 건으로 모든 사용자가 탈퇴하고 언론에 대서특필.

**리스크 수준:** 🔴 High  
**대응책:**
- 데이터 최소 수집 원칙: 절감 계산에 필요한 범위만 저장
- 서버 저장 없이 클라이언트 로컬 처리 가능한 데이터 분리
- ISO 27001 또는 ISMS 인증 로드맵 수립
- 침해 사고 대응 매뉴얼 사전 작성

---

## 💀 시나리오 5: 협상 코치 기능의 효과 미흡

**가상의 실패 원인**  
AI가 생성한 협상 스크립트를 실제로 사용해봤지만  
통신사 상담원이 "그런 혜택 없습니다"로 일관하여 효과가 없었다.  
사용자 신뢰 하락 → "결국 쓸모없는 기능"으로 인식.

**리스크 수준:** 🟡 Medium  
**대응책:**
- 실제 협상 성공 사례 데이터 수집 및 스크립트 지속 업데이트
- 특정 통신사·시기·상황별 성공률 표시 ("이 방법 성공률 68%")
- 사용자 피드백 루프: "성공했어요 / 실패했어요" 리포트 수집
- 커뮤니티 기능: 성공한 협상 팁 공유

---

## 💀 시나리오 6: 가족 구성원 동의 문제

**가상의 실패 원인**  
가족 통합 대시보드를 만들려면 모든 구성원이 앱에 가입해야 했다.  
배우자가 자신의 소비 내역 공유를 거부하거나,  
자녀 데이터 입력을 꺼리는 경우가 많아 "가족 통합"이 유명무실화됨.

**리스크 수준:** 🟡 Medium  
**대응책:**
- 1인 사용도 충분히 가치 있는 기능 설계 (가족 연동은 선택사항)
- 카테고리별 공개 범위 설정 ("통신비만 공유", "전체 공유" 등 권한 세분화)
- 가족 구성원 초대를 게임화: "우리 가족 절감 목표 달성 챌린지"

---

## 💀 시나리오 7: 대형 핀테크의 유사 기능 출시

**가상의 실패 원인**  
출시 6개월 후 토스·카카오페이가 유사한 "고정비 진단" 기능을 내장.  
기존 사용자 기반과 연동 데이터 우위에 눌려 Save & Grow는 도태됨.

**리스크 수준:** 🟡 Medium  
**대응책:**
- 속도: 대형 플랫폼이 따라오기 전에 사용자 습관 형성 완료
- 깊이: 단순 진단이 아닌 "투자 연결 + 협상 코치"는 단기간에 복제 불가
- 커뮤니티: 사용자 생성 콘텐츠 (협상 성공 사례)는 네트워크 효과로 방어
- 파트너십: 보험사·통신사와 제휴를 통한 독점 데이터 확보

## ✅ 프리모텀에서 얻은 핵심 교훈

1. **법적 검토가 가장 먼저다** — 투자 기능은 금융당국과 선제적으로 협의
2. **데이터 연동이 서비스의 생사를 결정한다** — 마이데이터 API를 핵심 기술 자산으로
3. **첫 3분이 전부다** — 온보딩에서 즉각적인 가치를 보여주지 못하면 이탈
4. **"투자 연결"이 진짜 해자다** — 경쟁사가 쉽게 복제 못하는 핵심 차별점 강화

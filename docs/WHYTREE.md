# WHYTREE
## 왜 우리는 이것을 만드는가 — 이유의 나무

Simon Sinek의 Golden Circle(WHY → HOW → WHAT) 구조를 기반으로,  
우리 프로젝트의 존재 이유를 뿌리부터 가지까지 추적한다.

---

## 최상위 WHY (Root)

### WHY-0: 러너는 자신에게 맞는 신발을 고르는 데 불필요하게 많은 시간과 돈을 낭비하고 있다

러닝화를 잘못 골랐을 때의 결과는 단순히 "별로인 신발"이 아니다. 발 통증, 부상, 러닝 포기로 이어질 수 있다. 평균 러닝화 가격이 10만~25만 원인 상황에서, 시행착오는 비용이 크다.

---

## 1단계 WHY 분해

### WHY-1A: 현재 추천 시스템이 개인 취향을 반영하지 못한다
- 무신사, SSG, 네이버 쇼핑의 기본 정렬은 "추천순", "인기순", "판매순"
- 이 지표들은 대중의 평균이지, 개인의 최적이 아니다
- 나이키 에어 줌 페가수스가 1위라도, 발볼이 넓은 사람에겐 맞지 않을 수 있다

### WHY-1B: 소비자가 스스로 판단하기 위한 정보가 분산되어 있다
- 유튜브 채널마다 다른 기준으로 러닝화를 평가한다
- 카페/블로그 리뷰는 개인 경험 위주이며 비교 어렵다
- 브랜드 공식 사이트의 스펙 설명은 소비자 언어가 아닌 기술 언어다

### WHY-1C: 리뷰 플랫폼의 평점이 변별력을 잃었다
- 무신사 러닝화 카테고리 상위 제품 대부분이 4.5~5.0점
- 0.1점 차이가 의미하는 바를 소비자가 해석할 수 없다
- 부정적 리뷰는 묻히고, 긍정 편향이 심하다

---

## 2단계 WHY 분해

### WHY-2A (←1A에서): 왜 개인화 추천이 없는가?
- 러닝화는 SKU 수가 매우 많고, 속성이 복잡하다 (무게, 쿠션, 드롭, 아웃솔 소재, 발볼 등)
- 기존 협업 필터링(Collaborative Filtering)은 "비슷한 유저가 산 것"을 추천 → 콜드 스타트 문제
- 사이즈, 발 유형, 러닝 거리, 지면 유형 등 러닝화 선택에 필요한 입력 변수가 일반 쇼핑몰 UX에서 수집되지 않음

### WHY-2B (←1B에서): 왜 리뷰 정보가 구조화되지 않았는가?
- 리뷰는 자유 텍스트 형식 → 기계가 읽기 어렵다
- 한국어 NLP는 영어 대비 처리 도구가 제한적이었다
- 이커머스 플랫폼 입장에서 리뷰 분석은 핵심 사업이 아니어서 투자 우선순위가 낮았다

### WHY-2C (←1C에서): 왜 평점이 변별력을 잃었는가?
- "좋아요/나빠요"라는 이진 판단이 복잡한 제품 속성을 하나의 숫자로 압축하기 때문
- 리뷰어마다 중요하게 생각하는 기준이 다르기 때문 (쿠션 중시 vs 무게 중시)
- 플랫폼이 긍정적 리뷰를 더 노출시키는 알고리즘을 운영하기 때문

---

## 3단계 WHY 분해

### WHY-3A (←2A에서): 왜 속성 기반 입력을 수집하지 않는가?
- 기존 UX에 "발 유형", "러닝 목적" 등의 온보딩 플로우가 없다
- 소비자가 자신의 발 유형을 알지 못하는 경우가 많다
- 플랫폼이 추가 데이터 수집보다 판매 전환율 최적화에 집중하기 때문

### WHY-3B (←2B에서): 왜 한국어 리뷰 분석이 어려운가?
- 한국어는 교착어 특성상 어간 분리, 조사 처리가 복잡하다
- "예쁘다", "이쁘다", "스타일리쉬하다"는 동의어지만 표면 형태가 다르다
- "디자인은 좋은데 무게는 별로"처럼 하나의 문장에 여러 속성과 감성이 혼재한다

### WHY-3C (←2C에서): 왜 속성별 평점이 존재하지 않는가?
- 속성별 평점 시스템은 리뷰 작성 UX를 복잡하게 만들어 리뷰 수를 줄일 위험이 있다
- 플랫폼이 정의한 속성과 소비자가 실제로 중요하게 여기는 속성이 다를 수 있다
- 사후에 기존 리뷰에서 소급 추출하는 기술이 필요한데, 이를 구현한 플랫폼이 없다

---

## HOW: 우리는 어떻게 해결하는가

WHY 트리의 뿌리들로부터 올라온 문제 구조를 바탕으로, 우리의 접근법을 설계했다.

```
문제 구조                           우리의 해결책
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
한국어 NLP 어려움          →  OKT 형태소 분석기로 어간 정규화
동의어 처리 불가           →  OKT 원형 복원 ("이쁘다" → "예쁘다")
감성 정보 부재             →  KNU SentiLex 감성 사전 적용
문맥 오염 (리뷰 전체 점수) →  KSS로 문장 단위 분리 후 속성별 분석
속성 미구조화              →  6대 속성 분류 체계 정의
유사 제품 추천 부재        →  속성 프로필 코사인 유사도 기반 추천
```

### HOW-1: 문장 단위 속성 감성 분석 파이프라인

```
리뷰 텍스트 입력
    ↓
KSS 문장 분리기
    ↓
OKT 형태소 분석 (어간 복원, 품사 태깅)
    ↓
속성 키워드 감지 (무게/쿠션/디자인/가격/발볼/착용감)
    ↓
KNU SentiLex 감성 점수 부여
    ↓
속성별 선호도 점수 집계
    ↓
제품 프로필 벡터 생성
```

### HOW-2: 유사도 기반 추천

- 각 제품의 속성 벡터를 생성
- 코사인 유사도로 제품 간 거리 계산
- 사용자 관심 제품 대비 유사도 Top-N 추천
- 무신사 플랫폼의 카테고리 목록 및 제품 상세 페이지에 통합

### HOW-3: 사용자 인터페이스 통합

- 카테고리 목록 페이지: 무게감/쿠션감/발볼넓이 속성 필터 추가
- 제품 상세 페이지: "비슷한 아이템 추천" 섹션에 유사 이유 태그 표시
- AI 추천 모달: 자유 텍스트 입력으로 개인화 추천 제공

---

## WHAT: 우리가 만드는 것

최종적으로 우리가 만드는 것은 세 가지다:

### WHAT-1: 리뷰 기반 제품 속성 프로필 데이터셋
- 무신사 러닝화 전 제품에 대해 6대 속성(무게감, 쿠션감, 디자인, 가격, 발볼, 착용감) 선호도 점수 산출
- 레이더 차트로 시각화된 제품 프로필

### WHAT-2: 속성 기반 유사 제품 추천 시스템
- "디자인이 비슷해요", "쿠션감이 비슷해요", "발볼 넓이가 비슷해요" 등 이유를 설명하는 추천
- 품절/단종 시 대체 제품 자동 추천으로 이탈 방지

### WHAT-3: 무신사 플랫폼 통합 AI 추천 UX
- ✨ AI 추천 버튼을 통한 자유형 텍스트 추천 인터페이스
- 속성 필터를 활용한 정밀 탐색 경험
